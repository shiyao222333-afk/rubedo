# Rubedo · 凝华 — 项目蓝图

> 蓝图 = 项目宪法。只写方向和原则，不写怎么做。

---

## 愿景

一个副业操作系统——打开就是一个时间日程表，每个时间段挂着今天要干的事。点进去就是对应的 SOP 流程页，能自动的自动、该手动的保留。

最终方向：每天打开凝华，它告诉你今天做什么、该怎么做，你只管执行和拍板。

---

## 在 OpusMagnum 五器工坊中的角色

```
Nigredo(外部采集) → Albedo(验真假) → Citrinitas(存知识) → Rubedo(赚钱执行)
     ↑                                                         │
     └────────────── Rubedo 调用 Nigredo API 获取数据 ─────────┘
```

凝华是变现的"最后一公里"。它从 Citrinitas 取知识，从 Nigredo 获取外部数据（群消息、爬虫结果），用自动化工具帮你执行，执行结果回流到 Citrinitas——下次更聪明。

**分工明晰**：Nigredo 负责"看到外面的世界"（企微监控、爬虫、数据采集），凝华负责"用好知识赚钱"（SOP 执行、时间审计、智能辅助）。

---

## 核心原则

1. **已跑通的优先优化** — 酷家乐 SOP 已经跑通在赚钱，重点不是"能不能做"，而是"能不能更快更省力"。新 SOP（如小红书）才遵循"先手动跑通验证"。老 SOP 从 100 分到 120 分，比新 SOP 从 0 到 1 更优先。
2. **逐步自动化** — 跑通的环节逐个替换成自动，没跑通的环节先留着手动做。不用一步到位追求全自动。
3. **一个 SOP 一个页面** — 酷家乐接单是一个页面，小红书选品是另一个页面。点进去就是这条 SOP 的完整流程，不是散落一地的工具。
4. **最少点击原则** — 打开就能用，不填表、不配置。时间记录点两下就该开始，需求确认客户说完自动出清单。
5. **数据回流** — 每单的执行数据自动存回 Citrinitas。接 5 单后，AI 比你更清楚你的时薪和客户偏好。
6. **采集交给 Nigredo** — 凡是"从外部获取数据"（企微群消息、爬虫、数据抓取），都走 Nigredo API。凝华不自己写采集逻辑，只负责调用。

---

## 当前重心

**酷家乐 SOP 优化 + 平台搭建**。SOP 已经跑通，核心空闲时间做三件事：

- 主界面架子（时间日程视图，能看到今天要干什么）
- 把酷家乐 SOP 搬进凝华（每个环节标注"手动/自动"，逐个替换成自动）
- 时间审计（量化优化效果：这次省了多少分钟）

自动化采集（企微群消息监控、自动回复）交给 Nigredo（馏析）负责。凝华通过调用 Nigredo API 获取消息和触发回复。

小红书带着做——有新想法就加个页面，但重心不偏移。

> 📌 **状态（2026-07-07）**：v0.4 地基（SQLite 数据层 + 主界面架子）已完成并通过多轮代码审查（见 `docs/REVIEW_T3_FINAL.md`、`docs/ADR-001-sqlite-migration.md`）。重心从"搭地基"正式转向**业务功能验证**——先把时间审计跑通、再把酷家乐 SOP 的环节逐个从手动切成自动。

---

## 边界

- 不做通用的接单平台（专注你自己的副业）
- 不做面向多用户的 SaaS（单人工具）
- 不替你做决定（辅助执行，你拍板）
- 不做复杂工作流引擎（SOP 页面的环节顺序是固定的，用"手动/自动"切换就够了）
- 首版不做移动端（桌面工具）

## 验收标准

| # | 标准 | 怎么算通过 |
|---|------|-----------|
| 1 | **酷家乐 SOP 搬进凝华** | 原来手动能跑通的流程，在凝华里照样跑得通。至少一个环节从手动切成了自动，时间确实省了。 |
| 2 | **时薪可见** | 每周自动算出酷家乐时薪，知道每单赚多少、哪些环节耗时最多。 |
| 3 | **越用越聪明** | 接 5 单后，AI 需求确认清单准确率明显提升。 |
| 4 | **手动/自动切换** | 酷家乐 SOP 中任意一个环节可以从"手动做"切换成"自动执行"，互不影响。自动执行的底层采集能力由 Nigredo 提供。 |
