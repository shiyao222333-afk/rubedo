# BUGLOG-calendar-blank-0706-FIXED

> 元数据：项目=rubedo(凝华) / 跨度=2026-07-05晚~07-06 / 总轮数=19(方案1~19) / 触发=同症状连续≥3轮未过验收+用户显式"记录所有失败经验" / 最终提交=6869719 / 流程依据=bug-triage skill
> 一句话背景：日历网格高度撑不满容器，底部长期留空白；试了19轮才锁定"工具条固定360px"根治。

## 1. 现象
- 技术表现：DayPilot 周视图 `#calendar` 网格高度 < 容器；时间轴下方、及日历与底部 `#detail-panel` 之间长期出现空白；窗口态与最大化态的空白大小不一致。
- 用户原话：「从昨晚到现在日历的高度就没有变高哪怕一次」「窗口状态没有空白了，但是最大化还是有空白」

## 2. 调试历程（证据链）

### 阶段一：误以为能改 DayPilot 自身高度撑高网格（方案 1~10，约前 10 轮）
- 假设：改 `cellHeight` / `height` 配置能让日历网格撑满容器。
- 验证方法：改 DayPilot 配置 + 诊断量 `#calendar` 渲染高度。
- 诊断证据：日历高度恒为 `580px` 不变（DayPilot `height:"auto"` 只在 `dp.init()` 算一次，并给 `#calendar` 写内联 `height:580px !important`）。
- 结论：❌ 排除 — auto 模式不随 resize 重算；且用户明确禁止改 DayPilot 高度配置。

### 阶段二：底部面板覆盖 + 动态测量（方案 11~17）
- 子方向 2a（方案 11，9fa4b29）：底部面板覆盖日历底 — 方向对，但不解决"两态空白不同"。
- 子方向 2b（方案 12）：`dp.update()` 触发重绘 + CSS `!important` 对抗 DayPilot 内联 `!important`。
  - 诊断证据：内联 `!important` 优先级高于样式表 `!important`，覆盖失败。
  - 结论：❌ 排除。
- 子方向 2c（方案 13，e62e6b4）：用 `getBoundingClientRect` 量"容器高−渲染高"算空白。
  - 诊断证据：空白恒≈0（被内联 `!important` 钉死，测不到真实底边缘差）。
  - 结论：❌ 排除 — 基准错，应量"日历底−面板顶"。
- 子方向 2d（方案 14~17，580de65/880a3ef/350cc78/080a482/ef63330）：resize 后 `dp.update()` + 双 rAF 重测 + ResizeObserver。
  - 诊断证据：面板高度跟窗口变（窗口 280 / 最大化 332）→ 两态空白大小不同；resize 后 DayPilot 异步重排 `#calendar` 底边缘，JS 测旧位置盖过去后 DayPilot 又推下 → 残留空白（"看起来变小了但还在"）；`.main-layout{height:100vh}` 没减导航栏高度 → 整体溢出视口 84px → 底部面板底边被裁到屏外。
  - 结论：❌ 排除 — 测量引发竞态/振荡，且两态不一致。
- 子方向 2e（方案 16 根治点，080a482）：修溢出 84px（`.main-layout` 减导航栏高度）。
  - 诊断证据：溢出消失，但空白仍来自面板高度跟窗口变（见 2d）。

### 阶段三：放弃测量，锁定固定值（方案 18~19）✅
- 方案 18（0bed156）：`--panel-h` 锁定，仅最大化时测量一次并缓存 localStorage。
  - 诊断证据：窗口态沿用 332、最大化测出 332，但根源仍是"依赖测量"，两态一致性脆弱。
  - 结论：⚠️ 部分正确 — 方向对（锁变量），但仍靠测量。
- 方案 19（619c7a6 改 350 → 6869719 改 360）：彻底删除测量逻辑，直接写固定常量 `TARGET_PANEL=360`，窗口态/最大化态都强制 `--panel-h:360px`。
  - 诊断证据：窗口态 `#detail-panel` 高度=360px / 最大化态=360px / 可见空白=0px。
  - 结论：✅ 确认。

## 3. 最终方案
- 关键改动：
  - `pages/index_page.py`：`#calendar { bottom: var(--panel-h, 360px) }`、`#detail-panel { height: var(--panel-h, 360px) }`（默认 360）。
  - `static/init.js`：删除 `getCalBottom` / `isMaximized` / 测量 + `localStorage` 缓存整套逻辑；改为直接 `document.documentElement.style.setProperty('--panel-h', '360px')` 并清掉旧缓存。
- 为什么有效：两态高度由同一 CSS 变量强制一致 → 窗口态上方日历区被压缩、底部工具条纹丝不动；全程未修改 DayPilot 的 `cellHeight` / `height` 配置（遵守用户硬约束）。
- 验证：诊断达标（见阶段三方案 19 证据）。

## 5. 教训（每条绑定具体证据）
1. 同方向连败≥3轮→换思路（PM 重试≤3）；本案例拖到第10轮才彻底转向（源于阶段一诊断恒580px却仍试）。
2. 不猜第三方库内部逻辑；查文档 + 量真实 DOM（`getBoundingClientRect`）拿证据（阶段一/二c）。
3. 测空白用"真实边缘差"（日历底−面板顶），不用"容器高−渲染高"（阶段二c：被钉成≈0）。
4. CSS `!important` 干不过内联 `!important`（阶段二b 证据）。
5. 绝对定位/全屏必须减导航栏高度，否则溢出裁切（阶段二e：溢出 84px）。
6. 测量引发竞态/振荡时，锁定固定常量优于反复测量（阶段二d→三：竞态残留空白）。
7. 验证走诊断工具（🔧 按钮一键复制纯文本），绝不 console.log（阶段二g）。
