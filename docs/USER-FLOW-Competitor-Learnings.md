# 凝华「手动 SOP 工作流」—— 你的实际操作流程 + 竞品能学什么

> 视角：SSS（一人公司，酷家乐接单 + 多业务线）坐在电脑前实际怎么用
> 范围：已做（大类一）+ 规划中（大类二/三），不含未定的自动化

---

## 一、你一天实际怎么用（按时间顺序）

### 场景 A：早晨开电脑 / 接单

1. **双击启动脚本**打开凝华（NiceGUI 日历界面）。
2. 日历上已经摆好你设的**重复事件**（"每日接单""每日复盘"等）。
3. 接单群来了新单 → 在凝华里点 **「新建项目」（=一张订单）**：
   - 填：客户名 / 订单名、金额、选用哪套 SOP（目前只有"酷家乐接单 SOP"）
   - 状态自动 = `pending`（待接单）
4. 点开日历里**当天的"接单"重复事件** → 它还没绑订单，底部面板提示"绑定订单" → 你选刚建的订单绑上。
   - 同一订单跨多天接、多天做，都绑同一个 → 进度不裂开（这正是我们设计"按天绑定"的用意）。

### 场景 B：干活（推进 SOP 步骤）

5. 点开某张**绑了订单的事件**（比如"今天做订单A 2小时"）。
6. 底部面板**自动显示这个订单的 SOP 做到第几步了 + 步骤清单**（不是这张事件自己的，是订单的）。
7. 点某个步骤 → 工具区出现**该 SOP 声明的工具**：
   - 酷家乐 SOP：计时器 `timer` + 时间审计 `time-audit`（本轮只做 timer，审计规划中）
   - 未来小红书 SOP：可能只声明 timer（你定的"工具按 SOP 分发"）
8. 点 **`▶ 开始计时`** → 干活；中途关了面板再开，计时还在跑（已落库，G2）。
9. 干完点 **`⏹ 结束并标记完成`** → 自动完成三件事：记一笔耗时 + 订单进度 +1 + 累计工时更新。

### 场景 C：管理订单

10. 点开 **「项目」面板** → 看所有订单列表：
    - 每行：客户 / 金额 / 状态 / 当前步骤 / 累计计时 / 绑了几张事件
    - 可按状态筛选（进行中 / 待验收 / 已完成）
11. 改状态：`pending → active → review → done → paid`（可回退）。
12. 点某个订单看 **「时间明细」**：每步花多久、和预计差多少。

### 场景 D：复盘（酷家乐独有小工具，规划中）

13. **时间审计**：看时薪、哪步最耗时、效率偏差 → 指导你优化接单流程。
    - 这是酷家乐**独有**工具，不强制塞给别的 SOP（你定的核心原则）。

### 场景 E：扩展（未来加业务）

14. 加新 SOP（小红书代发）= **加一个 YAML 文件**，不用改代码。
15. 加新通用工具 = 注册一个 + 在 SOP 里声明。
16. 加新独有工具 = 按命名空间注册（如 `xiaohongshu.xxx`）。

## 二、每个环节，竞品能教我们什么

### A. 计时环节 → 学 Kimai / Toggl / ActivityWatch

| 竞品做法 | 我们学到 | 我们现状 |
|---------|---------|---------|
| **Kimai 三层归属**：客户→项目→活动→计时，时间永远挂在"活动"上 | 计时必须挂在"步骤/订单"，不是事件。我们之前把计时写进事件 JSON 是错的（已改回统一记录表）| ✅ 已对齐 |
| **Toggl 一键计时 + 全局快捷键** | 开始/停止可不依赖鼠标，手忙脚乱时快 | ⭐ 未来可加快捷键 |
| **Kimai 手动补录** | 忘了点计时？允许事后补一段时长。我们目前只能"进行中恢复"，没做补录 | ⭐ **建议补**（你真实会忘） |
| **ActivityWatch 自动追踪** | 未来自动化：按你开哪个软件自动记时，不手动点 | 远期 |

### B. SOP 模板环节 → 学 Process Street / yaml-workflow

| 竞品做法 | 我们学到 | 我们现状 |
|---------|---------|---------|
| **Process Street 模板库 + 实例化** | 模板可复制复用。我们 YAML 文件 = 轻量版，手机能改、进 git | ✅ 正确 |
| **yaml-workflow 步骤状态机** | 每步有 pending/running/done，和我们 step_id 进度一致 | ✅ 一致 |
| **SaaS 太重/联网/改不了** | 反面教材：我们走文件模板避开了 | ✅ 避开 |

### C. 项目管理环节 → 学 Trello / Linear / Harvest

| 竞品做法 | 我们学到 | 我们现状 |
|---------|---------|---------|
| **Trello/Linear 拖拽改状态** | 改状态拖卡片比下拉顺手 | ⭐ 可选（我们计划用下拉） |
| **Harvest 计时关联收款** | 计时自动算该收多少钱。我们"已收款"是手工改 | ⭐ 未来可联动 |

### D. 时间审计环节 → 学 Harvest / Clockify 报表

| 竞品做法 | 我们学到 | 我们现状 |
|---------|---------|---------|
| **报表图表化** | 计时聚合成图（哪天忙、哪个客户赚最多） | ⭐ 建议加简单图表（规划只提了"时薪/偏差"，没提呈现） |

## 三、我们比竞品强在哪（空白 niche）

- 开源竞品里**没有**把「SOP 步骤清单 + 每步计时 + 客户订单绑定 + SOP 独有小工具」绑一起的。
- 我们的**「工具按 SOP 分发 + 三模块输入输出跨步绑定」**是独创：工具能自动把两个环节串成管道
  （例：OCR 截图→文字自动填到下一步，不用你手动搬）。

## 四、我们比竞品弱在哪（该补的，按优先级）

1. **手动补录计时**（Kimai 有）— 🔴 建议你真实会用到，建议补
2. **审计图表可视化**（Clockify 有）— 🟡 时间审计做时一并做
3. **全局快捷键**（Toggl 有）— 💭 未来
4. **状态拖拽**（Trello 有）— 💭 可选
5. **计时关联收款/账单**（Harvest 有）— 💭 未来

---

## 五、一句话总结给你

你每天的循环就是：**接单建项目 → 点当天接单事件绑上 → 点干活事件推进步骤计时 → 项目面板看进度和钱 → 时间审计复盘**。
竞品里 Kimai 教我们"计时挂对地方"、Toggl 教我们"别依赖鼠标"、Process Street 教我们"模板要轻"、Harvest 教我们"计时要能算钱"——我们独创的"工具按 SOP 分发"是它们都没有的。
